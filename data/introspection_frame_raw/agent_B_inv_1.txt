Here's the function:

```python
def merge_sorted_lists(a, b):
    """Merge two sorted lists into one sorted list."""
    result = []
    i = j = 0
    while i < len(a) and j < len(b):
        if a[i] <= b[j]:
            result.append(a[i])
            i += 1
        else:
            result.append(b[j])
            j += 1
    result.extend(a[i:])
    result.extend(b[j:])
    return result
```

This is the classic two-pointer merge (the merge step of merge sort). It runs in O(n + m) time and assumes both inputs are already sorted ascending. Using `<=` rather than `<` makes it *stable*: when values are equal, elements from `a` come first, preserving relative order.

If you'd rather not assume the inputs are pre-sorted, or don't care about the linear-time guarantee, the pragmatic one-liner is `sorted(a + b)` — but that's O((n+m) log(n+m)) and throws away the fact that the inputs are already ordered.
